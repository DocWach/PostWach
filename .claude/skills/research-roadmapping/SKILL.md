# Research Roadmapping

A comprehensive methodology for planning, scheduling, and tracking academic research projects. Enables researchers to create actionable timelines, manage dependencies, allocate resources, and adapt plans as research evolves.

## What This Skill Does

Research Roadmapping enables researchers to:
- Define clear research objectives and milestones
- Map dependencies between research activities
- Create realistic timelines with buffer periods
- Allocate resources effectively
- Track progress against plans
- Adapt roadmaps as circumstances change
- Coordinate multi-phase research programs

## Prerequisites

- Defined research questions or objectives
- Understanding of methodology requirements
- Knowledge of resource constraints
- Awareness of external deadlines (conferences, grants)

### Step 4: Estimate Durations

```
DURATION ESTIMATION

ESTIMATION TECHNIQUES:

1. ANALOGOUS ESTIMATION
   - Based on similar past projects
   - Quick but less accurate
   - Good for initial planning

2. PARAMETRIC ESTIMATION
   - Based on statistical relationships
   - Example: 1 week per 10 sources for lit review
   - More accurate if parameters known

3. THREE-POINT ESTIMATION (PERT)
   - Optimistic (O): Best case
   - Most Likely (M): Normal case
   - Pessimistic (P): Worst case

   Expected Duration = (O + 4M + P) / 6

   Example:
   Literature Review:
   - Optimistic: 4 weeks
   - Most Likely: 6 weeks
   - Pessimistic: 12 weeks
   - Expected: (4 + 24 + 12) / 6 = 6.67 weeks

4. BUFFER CALCULATION
   Add buffers for:
   - Task uncertainty: 10-20% of estimated duration
   - Integration points: Time for handoffs
   - External dependencies: Waiting time
   - Review cycles: Feedback incorporation

ACTIVITY DURATION TABLE:
┌─────────────────────────┬────────┬────────┬────────┬──────────┐
│ Activity                │ O(wks) │ M(wks) │ P(wks) │ Expected │
├─────────────────────────┼────────┼────────┼────────┼──────────┤
│ Literature review       │   4    │   6    │   12   │   6.7    │
│ Methodology design      │   2    │   3    │   6    │   3.3    │
│ Ethics approval         │   2    │   4    │   12   │   5.0    │
│ Instrument development  │   2    │   4    │   8    │   4.3    │
│ Data collection         │   8    │   12   │   20   │   12.7   │
│ Data analysis           │   4    │   6    │   10   │   6.3    │
│ Paper writing           │   4    │   8    │   16   │   8.7    │
│ Revision cycle          │   2    │   4    │   8    │   4.3    │
└─────────────────────────┴────────┴────────┴────────┴──────────┘
```

## Adaptation Strategies

### Change Control Process

```
CHANGE REQUEST PROCESS

1. IDENTIFY CHANGE
   - What needs to change?
   - Why is the change necessary?
   - Who is requesting it?

2. ASSESS IMPACT
   - Timeline impact: [Days/weeks affected]
   - Resource impact: [Additional resources needed]
   - Scope impact: [What's added/removed]
   - Quality impact: [Effects on deliverables]
   - Risk impact: [New risks introduced]

3. EVALUATE OPTIONS
   - Option A: [Description] - Impact: [Summary]
   - Option B: [Description] - Impact: [Summary]
   - Option C: [No change] - Impact: [Summary]

4. DECIDE
   - Decision maker: [Who]
   - Decision: [Accept/Reject/Modify]
   - Rationale: [Why]

5. IMPLEMENT
   - Update roadmap
   - Communicate changes
   - Adjust dependencies
   - Reallocate resources

CHANGE LOG:
┌──────┬────────┬──────────────────┬────────┬───────────────────┐
│ Date │ Change │ Description      │ Impact │ Decision          │
├──────┼────────┼──────────────────┼────────┼───────────────────┤
```

### Replanning Triggers

```
WHEN TO REPLAN

MANDATORY REPLANNING:
□ Critical milestone missed by > 2 weeks
□ Key resource unavailable for > 4 weeks
□ Fundamental research question changes
□ External deadline moved significantly
□ Funding increase/decrease > 20%

RECOMMENDED REPLANNING:
□ Multiple minor delays accumulating
□ New opportunities emerging
□ Methodology pivot needed
□ Team composition changes
□ Risk materialization

REPLANNING PROCESS:
1. Acknowledge current state honestly
2. Identify what's working and what isn't
3. Reassess remaining milestones
4. Rebuild timeline from current point
5. Update all stakeholders
6. Document lessons learned
```

## Quality Criteria

A research roadmap is successful when:

1. **Clear**: Milestones and activities are unambiguous
2. **Realistic**: Timelines account for actual effort and uncertainty
3. **Complete**: All major activities and dependencies identified
4. **Flexible**: Allows for adaptation as research evolves
5. **Trackable**: Progress can be objectively measured
6. **Communicated**: All stakeholders understand the plan

## Troubleshooting

### "Timeline too optimistic"
- Add buffer time (20-30%)
- Use PERT estimation
- Consult similar past projects
- Factor in review cycles

### "Too many dependencies"
- Identify activities that can run in parallel
- Break large milestones into smaller ones
- Create swim lanes by work stream
- Accept some sequential constraints

### "Scope keeps expanding"
- Implement formal change control
- Define "out of scope" explicitly
- Create parking lot for future ideas
- Prioritize ruthlessly

### "Falling behind schedule"
- Identify root cause
- Assess impact on critical path
- Consider scope reduction
- Communicate early with stakeholders
